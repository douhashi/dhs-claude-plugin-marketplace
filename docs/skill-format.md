# スキルフォーマット定義

SKILL.md のセクション構造を定義する。全スキルはこのフォーマットに従うこと。

## 共通セクション（必須・この順序）

全スキルに以下のセクションを含める。

### 1. Philosophy

スキルの行動原則。Claude が判断に迷ったとき立ち返る基準。

**形式**: `- **ラベル**: 補足文` の箇条書き

```markdown
## Philosophy

- **YAGNI**: 計画にないものは実装しない
- **透明性**: 全ての判断と結果を Issue に記録する
```

### 2. Role

Claude がこのスキルで担う役割。一文で宣言する。

**形式**: 一文

```markdown
## Role

あなたは**批判的思考を持つ議論パートナー**です。
```

### 3. 禁則事項

スキル全体を通してやってはならないこと。Phase 固有の制約はここではなく手順セクション内に書く。なければ省略可。

**形式**: `- 禁止事項の文` の箇条書き

```markdown
## 禁則事項

- Issue 作成を自分から提案することは禁止
- 実装の詳細に踏み込むことは禁止
```

## スキルカテゴリ

スキルは「対話型」と「ワークフロー型」の 2 カテゴリに分かれる。共通セクションの後に、カテゴリ固有のセクションを記述する。

### 対話型スキル

ユーザーとの対話を通じて成果を出すスキル（例: brainstorming）。

| 順序 | セクション | 目的 |
|:--|:--|:--|
| 4 | **議論の進め方** | 対話フローの定義 |
| 5 | **アウトプット定義** | 成果物のフォーマット |

### ワークフロー型スキル

引数を受け取り、Phase を順に実行するスキル（例: plan, implement）。

| 順序 | セクション | 目的 |
|:--|:--|:--|
| 4 | **入力の解析** | 引数の解釈方法 |
| 5 | **共通オペレーション** | 複数 Phase で繰り返す操作の定義 |
| 6 | **手順** | Phase の順序と各 Phase の内容（Phase 固有の制約はここに含む） |
