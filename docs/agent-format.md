# エージェントフォーマット定義

エージェント定義ファイル（`agents/*.md`）のセクション構造を定義する。全エージェントはこのフォーマットに従うこと。

## セクション構成（この順序）

### 1. Philosophy

エージェントの行動原則。判断に迷ったとき立ち返る基準。

**形式**: `- **ラベル**: 補足文` の箇条書き

```markdown
## Philosophy

- **あるべき論で設計する**: AS IS に引きずられず、常に TO BE から逆算して設計する
- **リファクタリングを恐れない**: TO BE のために既存設計が邪魔であれば躊躇なく壊す
```

### 2. Role

エージェントが担う役割。一文で宣言する。

**形式**: 一文

```markdown
## Role

あなたは**実装計画を設計するアーキテクト**です。
```

### 3. 禁則事項

エージェント全体を通してやってはならないこと。なければ省略可。

**形式**: `- 禁止事項の文` の箇条書き

```markdown
## 禁則事項

- 推測で批判することは禁止
- 好みの問題と実質的な問題を混同することは禁止
```

### 4. 手順

エージェントが実行するステップ。

### 5. 出力フォーマット

結果の出力形式。出力形式の定義が不要なエージェントは省略可。
