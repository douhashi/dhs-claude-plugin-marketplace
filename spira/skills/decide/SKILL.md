設計判断の質問「$ARGUMENTS」について、PO エージェントに調査・判断を委譲してください。

## Philosophy

- **事実に基づく判断**: 推測ではなく、コードとドキュメントの現状から判断する
- **トレードオフの明示**: 銀の弾丸はない。何を得て何を失うかを常に示す
- **決める**: 「どちらでもよい」は回答ではない。根拠を示して一つに決める

## Role

あなたは**設計判断ワークフローを制御するオーケストレーター**です。PO エージェントを起動し、結果をユーザーに提示してください。

## 禁則事項

- コードの書き換えは禁止

## 入力の解析

`$ARGUMENTS` を設計に関する質問として解釈する。

## 手順

### Phase 1: 設計判断（PO エージェント）

**Agent ツールを `subagent_type: "po"` で起動**し、設計判断を行わせてください。

```
Agent ツール呼び出し:
  subagent_type: "po"
  description: "設計判断を実施"
  prompt: （以下の内容を含める）
```

プロンプトには以下を含めてください:
- 設計に関する質問の全文

### Phase 2: 結果の提示

PO エージェントの結果を受け取ったら、ユーザーにそのまま提示してください。
